    La jerarquía tecnológica como orden sagrado

    No da lo mismo que todo sea lo mismo

    Hace tiempo que quería reflexionar a cerca de esto: todo es lo mismo pero parece diferente.

    Un ejemplo rotundo es la analogía entre el mundo del desarrollo web y la vida real. Ambos mundos son muy extensos, así que acotaremos el desarrollo web a las esferas que conozco más: HTML, CSS, Javascript.

    Un sitio web como éste en el que estás ahora es un sistema de códigos interrelacionados entre sí. Cada elemento vive anidado en un contexto mayor que lo contiene. Eso es el DOM.

    Todos los elementos del DOM vienen predefinidos por defecto, en el navegador, por una serie de propiedades o cualidades que son las que luego el programador usa / modifica para generar apariencias y comportamientos.

    Al igual que tú, cada elemento es una división de la existencia con unas cualidades.

    El programador está tan presente como el programa. La vida y la programación son tan "parecidos" porque están hechos de la misma sustancia.

    CSS se encargará de dar visibilidad, sensibilidad, tocar, rozar... Aquí es la separación, el olvido... Aquí es cómo es que las cosas lucen, son vistas. Gracias a la separación tiene valor el encuentro. Tiene que ver con el arquetipo 2. Aquí la mediación es hacia dentro, una de las partes absorviendo otras partes.

    Buena parte de la programación se construye a partir de renombrar de diferentes maneras las mismas cosas para generar tramas, discursos, automatismos. La recurrencia y el encapsulamiento generan riqueza semántica, más posibilidades, y eso es necesario, pero también generan una validación de las recurrencias como si fueran las cosas originales y eso hace que sea más dificil desentramar de dónde vienen realmente las ideas. Aquí la propuesta sería una reprogramación de nuestra psique más sencilla, una reprogramación que contemple y recuerde su propia estructura original.
