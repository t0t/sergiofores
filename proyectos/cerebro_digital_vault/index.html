Sistema de gestión de conocimiento personal que combina Obsidian como vault principal, Claude Code como interfaz de IA, y la filosofía 01234 como framework organizativo. Esta documentación explica su funcionamiento real y estructura actual.

¿Qué es CEREBRO-DIGITAL realmente?

Es un vault de Obsidian con aproximadamente 200+ archivos organizados según la filosofía 01234, que funciona como sistema unificado de gestión de información personal y profesional.

Ubicación: /Users/a01234/Administracion Personal/
Tamaño: ~50MB de notas, documentos y archivos multimedia
Último backup: Sincronizado automáticamente via iCloud
Acceso: Obsidian desktop + Claude Code via CLI

El sistema integra gestión patrimonial (información clasificada), proyectos de desarrollo web, investigación filosófica, y experimentación con IA generativa.

Cómo Funciona el Sistema

El vault opera en tres capas: Obsidian para visualización y organización, Claude Code para procesamiento e interacción, y filosofía 01234 para estructura conceptual.

Flujo de Trabajo Típico

1. Entrada de información: Via URL, PDFs, notas manuales, o input directo a Claude Code

2. Procesamiento inicial: Claude Code analiza el contenido y determina clasificación 01234

3. Organización: El archivo se coloca en la estructura correspondiente (notas/, PROYECTOS/, etc.)

4. Actualización de índices: Se actualizan Indice.canvas y archivos de referencia

5. Vinculación: Se establecen conexiones con contenido existente relevante

Protocolo de Sesión de Claude Code

Cada vez que Claude Code se conecta al vault, ejecuta automáticamente:

  1. Lectura de CLAUDE.md para recuperar contexto y reglas del sistema
  2. Verificación de cambios en archivos críticos desde la última sesión
  3. Actualización de memoria de contexto con información del proyecto actual
  4. Validación de integridad de la estructura del vault
  5. Preparación para tareas según el contexto de la sesión anterior

Sistema 01234: El Framework Organizativo

No es un concepto abstracto. Es la metodología práctica que organiza toda la información del vault. Cada archivo, carpeta y proceso se clasifica según estos 5 niveles.

Metamodelo de Procesamiento: Flujo completo desde la entrada hasta la salida, mostrando las fases de escucha, nodo (procesamiento central) y relación contextual.

Ontología Fundamental: Base conceptual que conecta la inmanencia con la vida a través del ser, la mente y la inferencia como procesos centrales.

Arquitectura Completa del Sistema 01234: Diagrama detallado que muestra la interconexión entre los 5 niveles, sus propiedades, facetas y formas de aplicación práctica. Este es el mapa de referencia que estructura todo el contenido del vault.

Nivel 0: Campo Cuántico

Aplicación práctica: Datos en bruto, URLs por procesar, PDFs sin categorizar, ideas iniciales en notas rápidas. Carpeta temporal donde llega información antes del procesamiento.

Ejemplo: Links guardados desde navegador, notas de voz transcritas, documentos escaneados pendientes de revisión.

Nivel 1: Modelo/Esencia

Aplicación práctica: Decisiones clave, principios fundamentales, reglas del sistema. CLAUDE.md pertenece a este nivel, así como documentos de políticas y directrices maestras.

Ejemplo: REGLAS_SISTEMA.md, políticas de inversión, principios de diseño de proyectos web.

Nivel 2: Análisis

Aplicación práctica: Desglose de información, análisis financieros, research de proyectos, comparativas de opciones. La mayor parte del contenido en notas/ opera en este nivel.

Ejemplo: Análisis de mercado inmobiliario, comparación de frameworks web, breakdown de filosofías.

Nivel 3: Comunicación

Aplicación práctica: Síntesis de información, documentación, canvas de Obsidian que conectan conceptos. Este nivel traduce el análisis en conocimiento estructurado.

Ejemplo: Indice.canvas, resúmenes ejecutivos, documentación de proyectos terminados.

Nivel 4: Materialización

Aplicación práctica: Proyectos ejecutados, código deployado, decisiones implementadas, documentos legales firmados. Todo lo que tiene impacto real en el mundo físico.

Ejemplo: Landing pages en producción, contratos firmados, inversiones ejecutadas, libros publicados.

Estructura Real del Vault

Organización de carpetas y archivos con su propósito específico y contenido actual.

📋 CLAUDE.md

Función: Memoria persistente de Claude Code. Contiene reglas de funcionamiento, convenciones del proyecto, y contexto crítico que se mantiene entre sesiones.

Contenido actual: ~500 líneas con instrucciones en español, preferencia de idioma, sistema 01234, reglas sobre Indice.canvas, gestión de archivos Canvas, y protocolos de trabajo.

🗺️ Indice.canvas

Función: Mapa visual principal del vault. JSON con nodos y aristas que representan las conexiones entre elementos principales del sistema.

Estado actual: Centro de control visual, requiere autorización explícita para modificaciones. PROTEGIDO: No modificar sin permiso directo.

📂 notas/

Función: Gestión operativa organizada por tema. Contiene el trabajo de análisis y seguimiento de diferentes áreas del proyecto.

Subcarpetas activas: proyectos/, finanzas/, filosofia/, tecnologia/, con nomenclatura: descripcion_clara_fecha.md

🚀 PROYECTOS/

Función: Desarrollo activo de proyectos. Principalmente desarrollo web y investigación filosófica con subcarpetas por proyecto específico.

Proyectos activos: desarrollo_web/sergiofores/, desarrollo_web/noeliarequena/, libro-01234/, cerebro_digital_vault/ (este proyecto)

📄 DOCUMENTOS/

Función: Documentos oficiales y legales. Información de alta sensibilidad con clasificación de seguridad estricta.

🔴 ULTRA-SECRETO: Jamás será compartido en internet ni subido a repositorios. Contiene información financiera y legal crítica.

💭 PROMPTS/

Función: Prompts diseñados para recrear el sistema completo desde cero. Garantiza continuidad del proyecto en diferentes contextos.

Estado: Integrado en Indice.canvas. Contiene instrucciones para replicar la estructura y funcionamiento del vault en nuevas sesiones.

Protocolos y Reglas del Sistema

Reglas operativas reales que gobiernan el funcionamiento del vault y la interacción con Claude Code.

🚫 Regla Crítica: Indice.canvas

Prohibido modificar Indice.canvas sin autorización explícita del usuario. Es un archivo especial que requiere permiso directo para cualquier cambio.

Origen: Establecida después de modificación no autorizada en sesión previa. Claude Code debe pedir permiso antes de cualquier cambio.

🗣️ Preferencia de Idioma

Siempre comunicarse en español. Es un requisito fundamental para todas las interacciones dentro del vault.

Excepción: Código y documentación técnica puede estar en inglés cuando sea estándar del sector.

📁 Convenciones de Nomenclatura

Archivos: descripcion_clara_fecha.md
Carpetas: MAYUSCULAS para categorías principales

Ejemplo válido: analisis_mercado_inmobiliario_2024-01-15.md
Carpeta válida: PROYECTOS/desarrollo_web/

🔄 Flujo de Estados 01234

Estado 0: Actualización de información via url, obsidian, notas, lectura de PDFs
Estado 1: Claude Code analiza y organiza rigurosamente, elimina lo no esencial
Estado 2: Desglose y análisis detallado del contenido
Estado 3: Visualización en Canvas, notas con enlaces
Estado 4: Almacenamiento en Obsidian, automatización con scripts Python

🛡️ Sistema de Clasificación

🟢 PÚBLICO: Filosofía, metodologías, proyectos web compartibles
🔴 ULTRA-SECRETO: Información financiera/legal, jamás compartida online

El sistema determina automáticamente la clasificación según el tipo de contenido y carpeta de destino.

📥 Entrada de Información

Fuentes de entrada reales:

  • URLs via Claude Code: WebFetch tool descarga y procesa contenido web
  • PDFs: Drag & drop a Obsidian, lectura via Read tool
  • Notas manuales: Creación directa en Obsidian
  • Prompts de consulta: Preguntas específicas al sistema
  • Documentos escaneados: OCR manual + integración
Ejemplo real: Usuario pega URL de artículo → Claude Code usa WebFetch → Extrae contenido → Clasifica según 01234 → Sugiere ubicación en vault

🔄 Procesamiento y Análisis

Flujo de procesamiento automático:

  • Clasificación 01234: Determine nivel conceptual del contenido
  • Análisis de relevancia: Conexión con contenido existente
  • Fragmentación semántica: Desglose en componentes clave
  • Detección de patrones: Identificación de temas recurrentes
  • Clasificación de seguridad: Público vs Ultra-Secreto
Tiempo real: ~2-3 minutos para análisis completo de documento medio (10-20 páginas)

💾 Almacenamiento y Estructura

Arquitectura de almacenamiento:

  • Obsidian vault: Almacenamiento principal en /Users/a01234/Administracion Personal/
  • iCloud sync: Respaldo automático en la nube
  • Estructura de carpetas: Organización según sistema 01234
  • Metadatos: Propiedades YAML en archivos .md
  • Índices visuales: Canvas files para navegación
Capacidad actual: ~200 archivos, ~50MB total, crecimiento sostenible hasta ~500MB

🔍 Sistema de Consultas

Métodos de búsqueda y recuperación:

  • Búsqueda por Claude Code: Grep, Glob, Task tools para búsqueda inteligente
  • Navegación visual: Obsidian graph view + canvas navigation
  • Búsqueda semántica: Conexiones por concepto y tema
  • Filtros de clasificación: Por nivel 01234, fecha, proyecto
  • Enlaces internos: Wikilinks [[]] para navegación rápida
Rendimiento: Búsqueda específica: ~30 segundos, búsqueda exploratoria: ~2-3 minutos

📊 Mejora Continua

Ciclo de optimización del sistema:

  • Revisión semanal: Validación de coherencia de estructura
  • Actualización de CLAUDE.md: Incorporación de nuevas reglas y patrones
  • Refactoring de contenido: Reorganización de información obsoleta
  • Optimización de índices: Actualización de Canvas y archivos de referencia
  • Análisis de patrones: Identificación de temas emergentes
Frecuencia real: Mantenimiento: 30 min/semana, optimización mayor: 2 horas/mes

🔗 Integración de Componentes

Cómo interactúan las herramientas:

  • Obsidian ↔ Claude Code: Lectura/escritura bidireccional de archivos
  • CLAUDE.md: Memoria persistente entre sesiones
  • Canvas files: Representación visual sincronizada
  • Sistema 01234: Estructura conceptual unificada
  • iCloud: Sincronización automática entre dispositivos
Punto crítico: Todo el flujo depende de la coherencia entre Obsidian y Claude Code

Limitaciones y Áreas de Mejora

Como cualquier sistema real, CEREBRO-DIGITAL tiene limitaciones conocidas y áreas que requieren desarrollo futuro.

⚠️ Limitaciones Técnicas

Dependencia de Claude Code: El sistema requiere sesiones activas de Claude Code para automatización
Memoria de contexto: Limitada por el contexto de sesión de Claude
Backup manual: Respaldos dependen de iCloud sync, no hay sistema automatizado propio

🔧 Áreas de Mejora Identificadas

Automatización de scripts: Los scripts Python en src/ requieren desarrollo
Integración entre herramientas: Conexión Obsidian-Claude Code podría ser más fluida
Escalabilidad: A medida que crece el vault, la organización manual se vuelve compleja
